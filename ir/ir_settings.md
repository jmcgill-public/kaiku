# IR Settings — KAAMOS Acoustic Spaces

*Primary tool: Voxengo Impulse Modeler → Pristine Space*
*Portability fallback: `hiljaisuus_ir.py` (Python/numpy, no proprietary tools required)*

Both IRs are Sabine-verified. Parameters below are physics-derived,
not guessed. See `forest_ir_spec.md` for the source reasoning.

## Voxengo Impulse Modeler

### IR 1 — Hiljaisuus Station Corridor

**Output file:** `hiljaisuus_station.wav`

| parameter | value | notes |
|-----------|-------|-------|
| Room length | 40m | corridor long axis |
| Room width | 12m | |
| Room height | 7m | |
| Reflection | 0.924 | amplitude; energy absorption 14.6% — metal |
| RT60 | ~2.2s | Sabine-verified for these dimensions |
| Output length | 4.0s | 2× RT60 — full tail |
| Sample rate | 48000 Hz | |
| Channels | Mono | one room, one channel |

Source and mic: place near one end of the long axis (position ≈ 5%
along the length). The 40m far-wall reflection arrives at ~221ms —
that late arrival is what makes this space feel like a corridor
rather than a room. If Impulse Modeler constrains source/mic to
normalized 0–1 positions, use 0.05.

### IR 2 — Voidborn Forest

**Output file:** `voidborn_forest.wav`

| parameter | value | notes |
|-----------|-------|-------|
| Room length | 18m | felt space between trunks |
| Room width | 14m | |
| Room height | 11m | canopy |
| Reflection | 0.636 | amplitude; energy absorption 59.6% — moss/pine |
| RT60 | ~0.62s | Sabine-verified for these dimensions |
| Output length | 2.0s | 3× RT60 — full tail captured |
| Sample rate | 48000 Hz | |
| Channels | Mono | |

Source and mic: center of space (position 0.5 on all axes).
Unlike the station, there is no "interesting end" — the forest
closes around you uniformly. The centered position gives the
most diffuse early reflection pattern.

**The HF rolloff that makes this a forest rather than a room:**
Impulse Modeler models specular reflections. The canopy absorption
and pine scatter that soften the high end above 4kHz will not be
captured by the model alone. Apply a gentle shelf in Pristine Space's
EQ section (if available) or with a buss EQ post-convolution:
high shelf, −3 to −6dB above 4kHz. Ear-check against Side B
birch flute material — it should feel outdoors, not reverberant.

## Loading in Pristine Space

Both IRs load as mono convolution sources. In the signal chain,
both run as parallel sends (not inserts) — see `signal_chain.md`.

The station IR is the primary room for the KAAMOS record.
The forest IR is the primary room for Éamonn an Chnoic (voidborn).
On Side B birch flute voices, the forest IR is available as a
secondary blend if the station reads too hard. Decision pending
ear-check.

Pristine Space gain staging: the IR normalization peak is 0.9.
Set the Pristine Space wet level to taste — start at −6dB wet
and adjust. The room should place, not wash.

## Voxengo Deconvolver — real IR capture (future)

Deconvolver measures real impulse responses from sweep tone recordings.
Workflow: play a sweep tone through a speaker in the target space,
record the result, run Deconvolver to extract the IR, export as WAV,
load into Pristine Space.

What this enables for the KAAMOS project:

**If you ever stand in a real metal corridor** matching the station
spec (40m+ length, hard surfaces, similar dimensions): capture it.
The synthetic IR is a model. A real corridor is the thing itself.

**If you stand by the Karelian lake with a laptop:** capture it.
The forest IR spec was derived from memory and physics. The real
Karelian forest IR would be the thing itself — moss underfoot,
pine canopy above, that specific quiet. Deconvolver handles the
measurement. You provide the forest.

The synthetic IR is not an apology for not having the real one.
It is the available tool used well. But the real one exists.

## Python fallback (`hiljaisuus_ir.py`)

For anyone without Impulse Modeler: the Python script generates
both IRs from the same parameters using the image source method
(numpy only, no proprietary tools). Output is float32 WAV,
Pristine Space compatible.

The script is in this repository. It is not the production tool.
It is the reason the IRs can be regenerated by anyone, anywhere,
with no software license.
